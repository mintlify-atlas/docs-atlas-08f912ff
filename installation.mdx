---
title: "Install json-everything .NET library packages from NuGet"
description: "Install one or more json-everything libraries via NuGet. Each library is published independently — install only what your project needs."
---

Json Everything libraries are published individually to NuGet. You install only the packages your project requires — there is no combined "everything" package. All libraries target .NET Standard 2.0 and .NET 8, and several include native AOT support.

## Available packages

### JSON Schema

| Package | Description |
|---|---|
| `JsonSchema.Net` | Core JSON Schema validation for drafts 6, 7, 2019-09, and 2020-12 |
| `JsonSchema.Net.Generation` | Generate schemas from .NET types using attributes and a fluent builder |
| `JsonSchema.Net.Generation.DataAnnotations` | Schema generation support for `System.ComponentModel.DataAnnotations` attributes |
| `JsonSchema.Net.DataGeneration` | Generate random JSON data instances that conform to a schema (powered by Bogus) |
| `JsonSchema.Net.Data` | A vocabulary for accessing instance and external data during validation |
| `JsonSchema.Net.ArrayExt` | A vocabulary for extended array validation rules |
| `JsonSchema.Net.OpenApi` | OpenAPI 3.1 vocabulary extension for JSON Schema |
| `JsonSchema.Net.Api` | ASP.NET Core middleware for automatic JSON Schema request body validation |

### Navigation and mutation

| Package | Description |
|---|---|
| `JsonPath.Net` | JSON Path querying per RFC 9535 |
| `JsonPatch.Net` | JSON Patch application and generation per RFC 6902 |
| `JsonPointer.Net` | JSON Pointer navigation per RFC 6901 and Relative JSON Pointer |

### Logic and templating

| Package | Description |
|---|---|
| `JsonLogic` | Declarative rule evaluation in JSON Logic format |
| `JsonE.Net` | JSON-e template evaluation for parameterized JSON structures |

### Utilities

| Package | Description |
|---|---|
| `Json.More.Net` | Extensions for deep equality, enum serialization, and AOT support |
| `Yaml2JsonNode` | Convert YAML documents (via YamlDotNet) to and from `JsonNode` |

## Installation

Replace `JsonSchema.Net` with the package name you need. All packages follow the same installation pattern.

<CodeGroup>

```bash dotnet CLI
dotnet add package JsonSchema.Net
```

```powershell Package Manager Console
Install-Package JsonSchema.Net
```

```xml PackageReference
<PackageReference Include="JsonSchema.Net" Version="*" />
```

</CodeGroup>

### Installing multiple packages

Install each package separately. For example, to use JSON Schema validation together with JSON Path querying:

<CodeGroup>

```bash dotnet CLI
dotnet add package JsonSchema.Net
dotnet add package JsonPath.Net
```

```xml PackageReference
<PackageReference Include="JsonSchema.Net" Version="*" />
<PackageReference Include="JsonPath.Net" Version="*" />
```

</CodeGroup>

<Tip>
  Pin to a specific version in production by replacing `*` with the version number, for example `Version="7.3.0"`. Check [NuGet](https://www.nuget.org/packages/JsonSchema.Net/) for the latest release.
</Tip>

## Framework support

All Json Everything libraries target two frameworks:

- **.NET Standard 2.0** — compatible with .NET Framework 4.6.1+, .NET Core 2.0+, Xamarin, Unity, and other platforms that implement the standard.
- **.NET 8** — optimized build taking advantage of current runtime features.

### Native AOT

`Json.More.Net` includes native AOT support. Other libraries in the suite depend on `System.Text.Json`, which has AOT support from .NET 7 onward. If you are publishing an AOT-compiled application, use the .NET 8 target and follow the standard `System.Text.Json` AOT guidance for your serialization contexts.

<Note>
  `JsonSchema.Net.DataGeneration` depends on [Bogus](https://github.com/bchavez/Bogus) for random data generation. Verify AOT compatibility with your specific Bogus version if you use this package in an AOT build.
</Note>
